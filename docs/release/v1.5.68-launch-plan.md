# SwarmClaw v1.5.68 Launch Plan

Date: April 23, 2026

## Release Theme

SwarmClaw v1.5.68 is the "Launch Week" release: make the product easier to share, easier to verify, and safer to promote in public.

## What Ships

- Launch Week Growth Sprint mission template for planning launch copy, Product Hunt and Show HN drafts, social/community updates, demo moments, daily feedback reports, and follow-up tasks.
- Next.js 16.2.4 in the app and docs site to address the current high-severity Next advisory range.
- Targeted dependency refreshes and npm overrides that remove high/critical production audit findings while avoiding a broad breaking audit-fix sweep.
- Package metadata alignment so `package.json` and `package-lock.json` agree on `1.5.68`.
- Electron release smoke builds restore host-architecture native modules after macOS multi-arch packaging, preventing the local checkout from being left in an incompatible native-module state.
- Updated README, website release notes, docs index, install pin, and public launch playbook.

## Positioning

Short version:

> SwarmClaw is a self-hosted control plane for autonomous AI agent teams: missions, memory, MCP tools, schedules, delegation, desktop installers, and 23 providers in one runtime.

Audience:

- Open-source AI agent builders who want local control instead of hosted-only agents.
- Founders and operators who want recurring autonomous work with budgets and reports.
- Devtools hackers who want to wire CLI agents, MCP servers, chat connectors, and memory together.

## Launch Channels

1. GitHub Release
   - Tag `v1.5.68`.
   - Let the existing release workflow publish npm, GHCR, and GitHub Release notes.
   - GitHub's generated release notes are useful because they include merged PRs, contributors, and changelog links: https://docs.github.com/en/repositories/releasing-projects-on-github/automatically-generated-release-notes

2. npm and Docker
   - npm package: `npm install -g @swarmclawai/swarmclaw@1.5.68`
   - Docker image: `ghcr.io/swarmclawai/swarmclaw:v1.5.68`
   - npm versions are immutable once published, so verify the version before tagging: https://docs.npmjs.com/cli/v10/commands/npm-publish/

3. Product Hunt
   - Prepare product tagline, maker comment, screenshots, demo video or GIF, and first-comment FAQ before launch day.
   - Official guide: https://www.producthunt.com/launch
   - Suggested tagline: "Self-hosted control plane for autonomous AI agent teams."

4. Hacker News
   - Use Show HN only if the demo/install path is ready the same day.
   - Keep it technical and useful; HN's guidelines explicitly discourage using the site primarily for promotion: https://news.ycombinator.com/newsguidelines.html
   - Suggested title: `Show HN: SwarmClaw - self-hosted runtime for autonomous AI agent swarms`

5. Community and Social
   - Discord: announce the release with the Launch Week template and ask for first-run feedback.
   - X/LinkedIn: lead with a 30-second demo and the desktop download link.
   - Reddit/Indie Hackers/devtool communities: post lessons learned, not pure promotion.

## Known Risks

- `npm audit --omit=dev --audit-level=high` passes, but still prints moderate transitive advisories in optional connector and indirect provider stacks. A full `npm audit fix --force` dry-run changes hundreds of packages and should be handled in a dedicated dependency-hardening pass.
- macOS desktop builds remain ad-hoc signed but not notarized, so the downloads page keeps the first-launch Gatekeeper guidance visible.
